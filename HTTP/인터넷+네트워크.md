# IP 프로토콜
* 지정한 IP 주소에 데이터 전달
* 패킷(Packet)이란 단위로 데이터 전달
    * 패킷에 출발지, 목적지, 데이터 등이 다 들어있다


# IP 프로토콜의 한계
* 비연결성
    * 패킷을 받을 대상이 없거나 서비스 불능 상태여도 패킷 전송
* 비신뢰성
    * 중간에 패킷이 사라지면?
        * 패킷을 전달하던 서버가 전달하던 등등 손실 가능
        * 그래도 손실됬는지 알 수 가 없다
    * 패킷이 순서대로 안오면?
        * 데이터를 나눠서 보내는데 경로가 다를 수 있음으로 순서가 바뀔 수 있다
* 프로그램 구분
    * 같은 IP를 사용하는 서버에서 통신하는 애플리케이션이 둘 이상이면?
